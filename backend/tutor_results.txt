--- Mode: Detailed ---
Backpropagation refers to a widely-used algorithm in neural network training that computes gradients for updating weights by applying the chain rule of calculus, propagating errors backwards through layers from output towards input. Here's how it works step by step during forward and backward passes within the context provided:

1. Forward Pass - The initial phase where inputs are passed layer-by-layer to reach a certain point in the network known as activation or logits, which represents an intermediate value before applying any nonlinearity (activation function). This results in predictions that will be compared against actual outputs later on for error calculation.

2. Loss Calculation - After obtaining these activations at each layer output from the forward pass and passing them through a loss function to evaluate prediction accuracy, we compute what is known as "loss" or difference between predicted values (outputs of our neural network) and actual target values provided in training data.

3. Backpropagation - It then starts calculating gradients needed for weight updates using the chain rule from calculus by propagating this loss backward through each layer, starting with the output:
   a. For every element calculated on its respective neuron's activation (output), we take note of how much that contributes to total network error—this forms an initial basis point in our gradient computation for weight adjustment purposes known as "error derivatives." These are determined by taking partial derivative with respect to the loss function, considering a specific output.
   
   b. Propagate these values backward through each hidden layer using their respective activation functions and transposed weights from subsequent layers—this is where applying chain rule comes into play in determining how much of error can be attributed directly to preceding neuron’s activations by multiplying the current derivative with previous weight connections (transpose).
   
   c. Continue this backward pass until reaching input layer, at which point we've obtained full gradients for each connection—these represent local sensitivities of network performance w.r.t changes in initial weights and biases; they are vital in directing how to update these parameters during learning process effectively.
   
4. Weight Update - After calculating the gradient flow through backpropagation, we use it alongside a specific algorithm (e.g., Gradient Descent) which will adjust each weight by subtracting or adding proportional amounts of gradients—this is done to minimize loss function iteratively in subsequent epochs until neural network's performance improves as evidenced through training progress visualized on the provided slides, like decreasing error rates over time.
